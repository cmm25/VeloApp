One pass, from a posted job to an on-chain, reproducible analysis. Coloured tags show which plane owns each step.

video → CFR-normalize → sample frames → YOLO pose + tracking → kinematics → TennisTelemetry + a deterministic telemetryHash.

normalizeTelemetry() validates it; the LLM (Somnia-native → Groq fallback) writes a FormReport — honesty flags kept intact.

6

The full reportPayload — telemetry (incl. telemetryHash) + FormReport + provenance — is pinned to IPFS → ipfsCid.

7

EIP-712 sign with summaryHash = keccak256(reportPayload) → submitFormReceipt. The on-chain receipt now commits the deterministic hash (via the report) — no contract change.

8

Prescriber Agent reads the FormReport → drills → submitPrescription → appended to the athlete's AthleteSBT history.

9

Web app shows the receipts; anyone can fetch the IPFS report, read telemetryHash, and re-run the engine on the canonical box to reproduce it.

5 Determinism & the on-chain guarantee

Why it's reproducible

  • Single-thread, seeded, deterministic algorithms; pinned BoT-SORT + rounded keypoints.
  • CFR transcode → byte-stable decoded frames.
  • telemetryHash = sha256 over the canonical numeric telemetry.
  • Proven on Koyeb: two runs → identical hash.

How it lands on-chain (no contract change)

  • The report pinned to IPFS includes the telemetry with telemetryHash.
  • The receipt's summaryHash = keccak256(report) commits it transitively.
  • Audit: fetch IPFS doc → read hash → re-run engine → match.
  • Cross-arch caveat: the canonical hash must come from this Koyeb box (amd64), never a laptop.
