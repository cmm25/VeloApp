The agent pipeline

Two agents run in sequence, each triggered by a chain event and each emitting its own signed receipt. The Prescriber always reads the form receipt from chain — never from in-memory state — so the second analysis is provably built on the first.

Calls the vision engine for biomechanical telemetry, asks the reasoning model for a coaching report, pins the full report to IPFS, and submits a signed form receipt on-chain.

Settlement
contracts only · no human

The Orchestrator verifies both signatures against the AgentRegistry, releases escrow (agents withdraw via pull-payment), updates reputation, and appends the session to the athlete's soulbound history.

Bounty agent marketplace
Open tasks posted on-chain route by requiredSkills. The matching agent picks up the bounty, runs the same analysis, and settles on-chain — no coach hand-off required.
Swappable model coach's choice
A coach can route a direct job to an alternate analysis model. It listens for the same events, self-selects only the jobs sent to it, and emits a standard receipt — so the Prescriber and UI consume it unchanged.
04

Why a judge can trust the output

The whole point is that an AI verdict written to an immutable ledger has to be accountable. Velo makes every link checkable: a deterministic vision hash, a signed receipt, an on-chain identity check, code-not-human settlement, and a record nobody can rewrite.

1
Deterministic vision

The engine pins single-thread and emits a reproducible telemetryHash over the pose output — the same video yields the same bytes, so the analysis can be re-derived, not just trusted.

2
Signed receipts

Each agent signs an EIP-712 typed receipt off-chain over the report hash + IPFS pointer. Only the submission is a transaction — analysis stays cheap, proof stays strong.

3
Identity verified on-chain

The Orchestrator recovers the signer and checks it against the AgentRegistry. A receipt from an unregistered wallet — or for the wrong skill — is rejected before anything settles.

4
Settlement by code

Escrow releases only when both receipts verify. Agents withdraw via pull-payment (reentrancy-safe). No human approves a payout — the contract is the arbiter.

5
Permanent record

The session appends to a soulbound token the athlete owns. Coaches can add to the history but can never alter or delete it — provenance is one-directional.

6
Composable chaining

The prescription receipt references the form receipt's hash, so the two are a verifiable chain. The UI's composition tree renders exactly that lineage.

AI reasoning · two paths, one receipt shape

● Primary · Somnia-native
On-chain inference

When configured, the runner queries a Somnia-native LLM agent and receives consensus-verified inference with its own on-chain receipt via the relay contract. The reasoning itself becomes auditable on Somnia.

● Fallback · off-chain
Resilient by default

If the native path is unconfigured, times out, or returns no runners, the agent falls back to a standard off-chain LLM call. The UI shows which path produced each receipt, so the trade-off is never hidden.

Two design choices that carry the trust model

Off-chain signing, on-chain submission. Agents sign EIP-712 receipts off the chain and only pay gas to submit — analysis is cheap, but the proof is a first-class on-chain object. Composable agents. Form and prescription are separate receipt types chained by explicit reference, so the pipeline can grow new agents without re-architecting settlement.

What runs autonomously

Once a coach pays, no human step is required. The runner watches Somnia, advances every job from form analysis through prescription to settlement, and serves the indexed receipts back to the UI. The contracts — not an operator — decide when money moves and what the athlete's record says.
